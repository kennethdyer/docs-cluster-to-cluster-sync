Overview
--------

Use this tutorial to manually install MongoDB Cluster-to-Cluster Sync
{+version+} on Linux using a downloaded ``.tgz`` tarball.

Cluster-to-Cluster Sync is available on select platforms.

Install Cluster-to-Cluster Sync
-------------------------------

Follow these steps to manually install Cluster-to-Cluster Sync from
the ``.tgz``.

.. procedure::
   :style: normal

   .. step:: Download the tarball.

      Download the Cluster-to-Cluster Sync ``.tgz`` tarball from the following
      link:

      |arrow| |mdb-download-link|

      a. In the :guilabel:`Version` dropdown, select the version of
         Cluster-to-Cluster Sync to download.

      #. In the :guilabel:`Platform` dropdown, select :guilabel:`macOS`.

      #. In the :guilabel:`Package` dropdown, select :guilabel:`shell (tgz)`.

      #. Click :guilabel:`Download`.

   .. step:: Extract the files from the downloaded archive.

      For example, from a system shell, you can extract the file using the ``tar`` command:

      .. code-block:: bash

         tar -zxvf mongosync-*.tgz

   .. step:: Ensure the binary is in a directory listed in your ``PATH`` environment variable.

      The ``mongosync`` binary is in the ``dist/`` directory of the tarball.
      You can either:

      - Copy the binary into a directory listed in your ``PATH``
        variable, such as ``/usr/local/bin`` (Update
        ``/path/to/mongosync/dist`` with your installation
        directory as appropriate)

        .. code-block:: bash

           sudo cp /path/to/mongosync/dist/* /usr/local/bin/

       - Create symbolic links to the binaries from a directory listed in
         your ``PATH`` variable, such as ``/usr/local/bin`` (Update
         ``/path/to/mongosync/dist/`` with your installation
         directory as appropriate):

         .. code-block:: bash

             sudo ln -s  /path/to/mongosync/dist/* /usr/local/bin/
