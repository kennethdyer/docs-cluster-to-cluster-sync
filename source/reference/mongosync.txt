=============
``mongosync``
=============

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

Synopsis
--------

The ``mongosync`` binary is the primary process used in Cluster to Cluster 
Synchronization.  It migrates data from one cluster to another and can keep the 
clusters in continuous sync.

Options
-------

Global Options
~~~~~~~~~~~~~~

.. option:: --cluster0 <URI>

   Sets the URI of the first cluster.

.. option:: --cluster1 <URI>

   Sets the URI of the second cluster.

.. option:: --verbosity <level>

   *Default*: ``INFO``

   Sets the verbosity level to use in log messages.

   This option accepts the following values:

   .. list-table::
      :header-rows: 1

      * - Level
        - Description
      * - ``TRACE``
        -
      * - ``DEBUG``
        -
      * - ``INFO``
        -
      * - ``WARN``
        -
      * - ``ERROR``
      * 
      * 
      * 
        -
      * - ``FATAL``
        -
      * - ``PANIC``
        -

.. option:: --logPath <DIR>

  Sets the path to the directory used to write log files

.. option:: --port

   *Default*: ``27182``

   Sets the port used by the HTTP API server.

.. option:: --id <ID>

   Sets the shard ID for the ``mongosync`` instance.

   Use this option when running multiple instances of ```mongosync``` on the
   same host.

.. option:: --config <filename>

   Sets the path to the configuration file.

.. option:: --version, -v

   Prints :binary:`mongosync` version information to stdout.

.. option:: --help, h

   Prints usage information to stdout.
